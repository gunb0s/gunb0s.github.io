8:Te54,<p>After years of constantly tweaking my development environment, I've finally settled on a setup that just works. Here's what I use daily and why these tools have stuck around.</p>
<h2>The Core Stack</h2>
<h3>Editor: VS Code + Vim Extension</h3>
<p>I've tried them all - Vim, Neovim, IntelliJ, Sublime Text. But VS Code with the Vim extension gives me the best of both worlds:</p>
<ul>
<li>Vim keybindings for editing efficiency</li>
<li>Rich ecosystem of extensions</li>
<li>Excellent debugging capabilities</li>
<li>Great Git integration</li>
</ul>
<p><strong>Key Extensions:</strong></p>
<ul>
<li>Vim (obvious reasons)</li>
<li>GitLens (git blame and history)</li>
<li>Thunder Client (API testing)</li>
<li>Error Lens (inline error display)</li>
</ul>
<h3>Terminal: iTerm2 + Oh My Zsh</h3>
<p>My terminal setup focuses on information density and speed:</p>
<pre><code class="language-bash"># My essential aliases
alias ll='ls -la'
alias g='git'
alias gst='git status'
alias gco='git checkout'
alias gp='git push'
alias gc='git commit -m'

# Docker shortcuts
alias dps='docker ps'
alias dcu='docker-compose up'
alias dcd='docker-compose down'
</code></pre>
<h3>Version Control: Git + GitHub CLI</h3>
<p>I've streamlined my Git workflow with these configurations:</p>
<pre><code class="language-bash"># Global Git config highlights
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.unstage 'reset HEAD --'
</code></pre>
<h2>Development Tools</h2>
<h3>API Development: Bruno/Postman</h3>
<p>For API testing and development, I use Bruno for personal projects and Postman for team work. Bruno's local-first approach appeals to me, but Postman's collaboration features are unmatched.</p>
<h3>Database: TablePlus</h3>
<p>Best database GUI I've used. Clean interface, supports multiple database types, and excellent query building capabilities.</p>
<h3>Containerization: Docker Desktop</h3>
<p>Essential for consistent development environments. My standard <code>docker-compose.yml</code> template:</p>
<pre><code class="language-yaml">version: "3.8"
services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=development
    volumes:
      - .:/app
      - /app/node_modules
</code></pre>
<h2>Productivity Boosters</h2>
<h3>Task Management: Notion</h3>
<p>I use Notion for everything:</p>
<ul>
<li>Project planning and tracking</li>
<li>Meeting notes and documentation</li>
<li>Personal knowledge base</li>
<li>Code snippets and learnings</li>
</ul>
<h3>Focus: Time-blocking + Pomodoro</h3>
<p>My daily schedule follows strict time-blocking:</p>
<ul>
<li>9-11 AM: Deep work (coding)</li>
<li>11-12 PM: Meetings/communication</li>
<li>1-3 PM: Deep work (coding)</li>
<li>3-4 PM: Learning/research</li>
<li>4-5 PM: Admin/planning</li>
</ul>
<h2>The Philosophy</h2>
<p>The key insight I've learned: <strong>tools should disappear</strong>. The best setup is one you don't think about. Every tool in my stack serves a specific purpose and has earned its place through daily use.</p>
<p>I've stopped chasing the latest tools and instead focused on mastering the ones I have. This approach has made me significantly more productive and less distracted by shiny new tools.</p>
<h2>What's Next?</h2>
<p>I'm currently exploring:</p>
<ul>
<li>Tmux for better terminal session management</li>
<li>Raycast as a Spotlight replacement</li>
<li>Linear for project management (might replace Notion)</li>
</ul>
<p>What tools have genuinely improved your development workflow? I'm always curious to hear what works for others!</p>
